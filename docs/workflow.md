## Development Pattern

1. **Start with behavior.** What should the system do? Write that down.
2. **Build the smallest version.** Get it working in hours, not days.
3. **Document as you go.** AGENTS.md files capture context.
4. **Ship early.** Real feedback beats hypothetical planning.
5. **Refactor with tests.** Behavior tests ensure nothing breaks.
