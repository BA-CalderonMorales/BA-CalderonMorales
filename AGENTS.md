# Agent Guidelines

This repo holds my GitHub profile README—a focused gateway into who I am and what I am building.

## Core Philosophy

This repository follows the [three-layer architecture](../../PHILOSOPHY.md):

- **Raw:** Source, assets, scripts
- **Wiki:** This AGENTS.md, docs/ folder
- **Schema:** [PHILOSOPHY.md](../../PHILOSOPHY.md) (workspace root)

## Intent

- Present a clear, minimal snapshot of current focus: agentic workflows, AI CLI tooling, developer environments.
- Avoid clutter. No decorative SVGs, contribution graphs, or badge walls.
- One featured project (Terminal Jarvis) with links; extended details live in docs/.

## Style

- Center only the name and tagline; left-align the rest.
- Short blurb per section, then bullet points.
- Plain Markdown with minimal HTML (alignment only).
- Tone: confident but understated. Show, do not shout.
- No emojis in root-level documentation.

## Structure

```
README.md          # Public-facing, slim
docs/
  philosophy.md    # How I think
  workflow.md      # How I work
  projects.md      # Extended project descriptions
  resources.md     # Guides and references
AGENTS.md          # This file
```

## Editing

- Keep updates atomic. One theme per commit.
- If adding a section, remove something else to preserve focus.
- Test links before pushing.
- Maintain sync with workspace PHILOSOPHY.md.
